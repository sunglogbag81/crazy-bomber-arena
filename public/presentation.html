Part 1 / Concept

발표용 프로젝트로 왜 게임인가?

좋은 발표용 프로젝트는 보는 사람이 바로 이해하고, 직접 해보고 싶고, 기술 설명까지 자연스럽게 이어져야 합니다.

몰입

보자마자 이해됨

이동, 폭탄, 폭발, 승리라는 규칙이 직관적입니다.

시연

분위기가 살아남

친구가 직접 들어오면 발표가 그냥 설명이 아니라 이벤트가 됩니다.

기술

설명할 게 많음

실시간 통신, 서버 상태관리, Canvas 렌더링을 한 번에 보여줍니다.

안전

솔로 모드 가능

네트워크나 인원 문제가 있어도 봇 상대로 바로 시연할 수 있습니다.

Part 2 / Gameplay

게임 규칙은 단순하지만, 시연은 강하다

Rules

기본 규칙

1이동하면서 폭탄을 설치한다
2폭탄은 십자 방향으로 터진다
3블록을 부수고 상대를 압박한다
4마지막까지 살아남으면 승리
Controls

조작

WASD / 방향키: 이동
Space: 폭탄 설치
모바일: 화면 버튼 지원
Part 3 / Architecture

핵심 구조: 클라이언트는 입력, 서버는 판정

Part 4 / Features

구현한 기능

Online

방 코드 대전

방을 만들면 4자리 코드가 생성되고, 친구는 같은 URL에서 코드로 입장합니다.

Battle

폭탄/폭발/블록

타이머 폭탄, 십자 폭발, 블록 파괴, 연쇄 폭발을 구현했습니다.

Solo

AI 봇 테스트

봇이 이동하고, 위험 지역을 피하고, 안전할 때 폭탄을 설치합니다.

Mobile

터치 조작

노트북이 없어도 휴대폰에서 방향키와 폭탄 버튼으로 플레이할 수 있습니다.

Deploy

리눅스 서버 실행

현재 서버에서 Node.js로 실행 중이고, 포트포워딩 후 외부 접속이 가능합니다.

Polish

발표 안정성

헬스체크, Dockerfile, Render 설정, 초대 문구 복사까지 포함했습니다.

Part 5 / Debug Story

개발 중 발견한 버그와 해결

Bug 1

폭탄이 다른 칸에 설치됨

캐릭터가 오른쪽 아래에 걸치면 폭탄도 오른쪽 아래 타일에 찍히는 문제가 있었습니다.

기존: Math.floor((px + TILE / 2) / TILE) 수정: Math.floor(px / TILE)
Fix 1

현재 서 있는 타일 기준으로 수정

타일 계산에서 불필요한 보정을 제거해서 폭탄이 플레이어 위치에 정확히 설치되게 했습니다.

중심 기준 좌표 계산 정리
발표 전 직접 플레이하면서 검증
Bug 2

봇이 자꾸 자살함

봇이 폭탄을 놓은 뒤 탈출 가능한 경로가 있는지 확인하지 않았습니다.

Fix 2

탈출 경로 검사 추가

폭탄 설치 전 BFS로 안전한 탈출 칸을 검사하고, 위험 지역에서는 회피 판단을 더 자주 하게 만들었습니다.

Part 7 / How to Join

접속 방법

같은 네트워크에서는 내부 IP로 접속하고, 외부 발표용은 공유기 포트포워딩 또는 도메인 연결을 사용합니다.

Local

같은 와이파이

http://192.168.0.4:3000
3000번 포트가 게임 서버
80번은 AdGuardHome이 사용 중
Online

외부 접속

http://도메인:3000
외부 3000 → 192.168.0.4:3000 TCP 포트포워딩
방 코드 공유 후 친구 입장
