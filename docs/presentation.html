01 / WHYCrazy Bomber Arena

왜 이 프로젝트를 만들었나?

발표용 프로젝트는 “보는 순간 이해되고, 직접 해보고 싶고, 기술 설명도 가능한 것”이 가장 강합니다.

🎮

게임이라 즉시 몰입된다

복잡한 설명 없이도 이동, 폭탄, 폭발, 승리라는 규칙을 바로 이해할 수 있습니다.

🌐

온라인 대전으로 임팩트가 있다

방 코드를 공유하면 여러 사람이 같은 게임 상태를 실시간으로 봅니다.

🤖

혼자서도 시연 가능하다

발표 중 네트워크나 인원 문제가 생겨도 봇과 솔로 테스트를 진행할 수 있습니다.

⚙️

기술 설명이 깔끔하다

클라이언트, 서버, 실시간 통신, 게임 루프를 한 프로젝트 안에서 설명할 수 있습니다.

발표자료1
02 / GAMECore Gameplay

게임 규칙

목표는 간단합니다. 폭탄을 설치하고, 블록을 부수고, 상대의 체력을 먼저 0으로 만들면 승리합니다.

조작

  • WASD 또는 방향키로 이동
  • Space로 폭탄 설치
  • 모바일은 화면 버튼 지원

승리 조건

  • 폭발에 맞으면 체력 감소
  • 마지막까지 살아남으면 승리
  • 라운드 종료 후 자동 재시작
발표자료2
03 / TECHArchitecture

기술 구조

핵심은 서버가 게임 상태를 관리하고, 클라이언트는 입력만 보내는 구조입니다.

1

입력

플레이어가 이동키나 폭탄 키를 누릅니다.

2

전송

클라이언트가 Socket.IO로 입력을 서버에 보냅니다.

3

계산

서버가 이동, 충돌, 폭발, 승패를 계산합니다.

4

동기화

서버가 모든 플레이어에게 최신 상태를 전송합니다.

서버 중심 구조를 선택한 이유

  • 모든 플레이어가 같은 게임 상태를 보게 만들기 쉽습니다.
  • 폭탄/폭발/승패 판정을 한 곳에서 처리해 버그를 줄입니다.
  • 나중에 랭킹, 방 목록, 아이템 같은 기능을 확장하기 좋습니다.
client input → server game loop → broadcast state → canvas render

04 / FEATURESImplemented

구현 기능

발표 시연에 필요한 핵심 기능은 모두 들어가 있습니다.

🏠

방 코드 시스템

방을 만들면 4자리 코드가 생성되고, 친구는 코드로 바로 입장합니다.

💥

폭탄과 폭발

폭탄 타이머, 십자 폭발, 블록 파괴, 연쇄 폭발을 구현했습니다.

🧱

맵과 충돌

고정 벽, 파괴 가능한 박스, 플레이어 이동 충돌을 서버에서 판정합니다.

🤖

AI 봇

솔로 테스트용 봇이 이동하고, 위험 지역을 피하고, 안전할 때 폭탄을 설치합니다.

📱

모바일 조작

스마트폰에서도 방향 버튼과 폭탄 버튼으로 시연할 수 있습니다.

🚀

배포 준비

Node.js 서버, Dockerfile, Render 설정, 헬스체크 엔드포인트를 포함했습니다.

발표자료4
05 / DEMOHow to Present

시연 순서

발표 때는 아래 흐름대로 하면 자연스럽고 안정적입니다.

추천 시연 루틴

  1. 게임 페이지를 열고 로비 디자인을 보여줍니다.
  2. 솔로 테스트를 눌러 봇과 즉시 플레이합니다.
  3. 폭탄 설치, 블록 파괴, 체력 감소를 보여줍니다.
  4. 다른 브라우저나 친구 휴대폰으로 방 코드 입장을 보여줍니다.
  5. 마지막으로 서버 구조와 Socket.IO 동기화 방식을 설명합니다.

발표 멘트 예시

“이 게임은 클라이언트가 직접 판정하지 않고, 서버가 모든 게임 상태를 계산합니다. 그래서 여러 명이 접속해도 같은 맵과 같은 폭발을 보게 됩니다.”

비상 플랜

인터넷이나 친구 접속이 불안하면 솔로 테스트 모드로도 전체 게임 흐름을 보여줄 수 있습니다.
